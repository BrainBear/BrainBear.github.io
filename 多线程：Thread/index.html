多线程：Thread

Posted on 2019-11-02

Thread 的生命周期

线程一共有 6 种状态，也就是它的生命周期变换如下：

stateDiagram-v2
    New --> Runnable 
    Runnable --> Blocked
    Blocked --> Runnable
    Runnable --> Waiting
    Waiting --> Runnable
    Runnable --> Timed_Waiting
    Timed_Waiting --> Runnable
    Runnable --> Terminated

  • New
    当线程创建成功，但是还没有启动之前，就是处于 New 状态

  • Runnable
    当 Thread 调用 start 方法之后，线程就处于 Runnable 状态，但是调用 start 方法并不代表线程马上就可以获取到 cpu 并且开始运行了，所以 Runnable 包含了就绪和正在运行两个含义，当正在运行的线程被 cpu 调度没有真正在执行的时候，它也是 Runnable 状态。

  • Blocked
    只有当线程进入 Synchronized 代码块并且没有获取到锁的时候就会进入 Blocked 状态。只有当线程获取到 Synchronized 的 monitor 锁之后才会恢复到 Runnable 状态

  • Waiting
    当线程遇到一些等待的方法时候，如果方法没有设置超时则进入 Waiting 状态，常见的有 Object.wait() 和 Thread.join() 等，对应的当调用 Object.notify() 之后线程会回到 Runnable 状态

  • Timed Waiting
    当线程遇到一些等待的方法时候，如果方法有超时则进入 Timed Waiting 状态，常见的有 Object.wait(time) 和 Thread.join(time) 等，恢复 Runnable 状态除了和 Waiting 的那些方法之外，等待的时间到了也会主动恢复

  • Terminated
    当线程运行完成才会进入 Terminated 状态，包括正常运行结束和异常结束

通过调用 Thread.getState() 方法可以获取到线程的当前状态。一般我们都把 Blocked（阻塞）、Waiting（等待）、Timed Waiting（计时等待）都称为阻塞状态。

怎么停止线程

启动一个线程是比较简单的，但是要怎么去停止一个线程呢？

stop

Thread 提供了一个 stop() 方法，但是这个方法太暴力了，会直接停止线程，并且释放所持有的锁，这样有可能会造成无法预估的后果，所以已经被弃用了。

Boolean 标志位

很多人喜欢用一个 Boolean 标志位来标识线程是否该取消操作，但是如果在线程进入阻塞状态的话，Boolean 标志位是不会生效的。

因为 BlockingQueue 的 take 方法会阻塞当前线程，所以即使 cancel 标志位被赋值为 true ，但是线程已经被阻塞了，没有执行到判断的逻辑，线程就这样卡死了。

interrupt

为了解决上面的问题，thread 提供了 interrupt() 方法。上面 BlockingQueue 的 take 方法会抛出一个 InterruptedException 异常，这个异常就是响应用户调用 interrupt() 方法抛出的，我们在处理这个异常的时候就可以执行相应退出线程的逻辑。

sleep

sleep 是我们比较常用的线程方法，他可以响应线程的中断，需要注意的是 sleep 不会释放持有的锁，包括 synchronized 和 lock 。

join

join 在前面演示 interrupt 的时候也用到了，它的作用就是在一个线程等待另外一个线程执行完毕，他也可以响应线程的中断。

yield

yield 比较少用，它的作用就是释放自己的 cpu 时间片，但是不保证让出的结果，有可能执行完之后 cpu 并没有让出，也有可能 cpu 让出之后马上又调度回来了，一般在一些 cas 操作中用到。
