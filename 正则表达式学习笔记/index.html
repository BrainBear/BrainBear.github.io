正则表达式

字符集合区间

在使用正则表达式的时候，会频繁地使用到一些字符集合，例如
数字：[0123456789]
大写字母：[ABCDEFGHIJKLMNOPQRSTUVWXYZ]
小写字母：[abcdefghijklmnopqrstuvwxyz]
字母：[ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz]

要写这么一长串是要死人了，所以正则表达式为了简化，提供了 -（连字符）来定义一个字符区间。

[0-9] 的功能和 [0123456789] 是完全一样的

同理：
大写字母：[A-Z] = [ABCDEFGHIJKLMNOPQRSTUVWXYZ]
小写字母：[a-z] = [abcdefghijklmnopqrstuvwxyz]
字母：[A-Za-z] = [ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz]

注意：不能使用 [A-z] 来匹配字母，因为字符区间匹配的其实是 ASCII 字符，你去查一下 ASCII 码表就会发现 Z 和 a 之间还有一些字符。

注意 2： - 连字符在正则表达式中是一个元字符，但是作为元字符它只能用在 [ 和 ] 之间，在字符集合以外的地方， -只是一个普通字符，只能和 - 本身匹配，所以在正则表达式中不需要对 - 转义。

匹配特定类型的字符

正则表达式中提供了一些特殊字符来匹配特定类型的字符，这些特殊字符都可以通过字符集合来实现，正则表达式是为了简洁而提供的，但是简洁也带来了可读性的问题，所以要不要用就见仁见智了。

元字符 描述
\s 匹配任何一个空白字符 等价 [\f\n\r\t\v]
\S 匹配任何一个非空白字符 等价 [^\f\n\r\t\v]
\d 匹配任何一个数字字符 等价 [0-9]
\D 匹配任何一个非数字字符 等价 [^0-9]
\w 匹配一个字母数字或下划线字符 等价 [0-9A-Za-z_]
\W 匹配一个非字母数字或下划线字符 等价 [^0-9A-Za-z_]

重复匹配

前面已经讲了很多匹配的方法，如匹配数字，字母等。但是他们都只能匹配一个字符，如果用前面的字符集合来匹配两个数字就会写成这样： [0-9][0-9] ，如果是匹配手机号码呢？也这样写的话太麻烦了。还有一种情况是匹配一种可能的情况，可能存在可能不存在的逻辑用前面讲的知识不能实现，需要用到下面讲到的重复匹配的知识。

匹配重复次数

前面讲的 * 和 * 都可以匹配连续多个字符，但是他们匹配的个数没有上限，如果我们要匹配手机号码，手机号码都是 11 位的，如果用 [0-9]+ 或 [0-9]* 都不正确，因为他们都可以匹配超过 11 位的连续数字。

为了解决这个问题，正则表达式提供了 { 和 } 让用户可以对字符的匹配字数进行更有效的控制。

表达式 描述
{n} 匹配 n 次
{n,m} 最少匹配 n 次，最多匹配 m 次
{n,} 至少匹配 n 次

例如想校验用户输入的手机号是否是 11 位的纯数字（中国手机号有特定的规律可以更精细地校验，这里只是简单举例）可以使用正则表达式：[0-9]{11}
