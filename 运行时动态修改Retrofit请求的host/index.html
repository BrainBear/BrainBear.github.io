运行时动态修改Retrofit请求的host

Posted on 2019-08-10

运行时修改 host

由于公司的服务接入了 CPS ，客户端不同的业务请求链接都需要在运行时动态设置，我们使用的请求库是 Retrofit，在网上找了一下相关的资料，确定使用 OkHttp 的拦截链来实现。

实现

之前来分析 OkHttp 的时候，发现他是通过一系列的拦截器来把整个网络请求响应串起来的，而且 OkHttp 也支持添加自定义的拦截器来实现特定的功能。而 Retrofit 也是依赖 OkHttp 的，所以我们可以通过添加拦截器来动态修改 Retrofit 请求的 host。

整体的思路是参考 JessYan 的，他开源了一个第三方库 RetrofitUrlManager ，这个库可以实现更强大的功能，但是我用不到这些功能，所以我就自己简单实现一遍。

思路

  1. 给同一类请求添加 header 标识
  2. 给 Retrofit 使用的 OkHttp client 增加拦截器
  3. 在拦截器中查询特定的 header ，查询到就替换指定的 host
