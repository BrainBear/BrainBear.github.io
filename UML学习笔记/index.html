UML学习笔记

Posted on 2018-07-22

UML是一种建模语言，简单来说就是一种图形标识法。为什么要学UML呢？是因为最近在看比较大的源码工程的时候，发现分析源码的时候比较吃力，从头到尾把源码理解读了一遍，读完发现虽然能理解大概，但是不能掌握全局的结构。在阅读一些优秀的博文的时候，发现都会贴上UML类图，一个图可以把核心的类相互的逻清晰地辑理出来。所以开始学习UML类图。

UML类图

UML有很多种图，这里只介绍类图。

类的属性和方法

类在UML类图中用内部分隔开的矩形表示，如下表示一个Student类：

最上面的矩形表示类名，第二个矩形表示属性，第三个矩形表示方法。

属性的表示方式是：可见性 名称 ：类型 [ = 缺省值]
方法的表示方式是：可见性 名称 [ ：返回值]

[ ]中的部分表示可选

属性和方法是可选的，一个类可以没有属性，也可以没有方法，也可以属性和方法都没有。

可见性

属性和方法最前面的符号表示可见性，有以下可见性

  • + : public
  • - : private
  • # : protected

所以上面的Student类图表示Student有两个public的属性name和age，一个public的方法display。

类之间的关系

泛化 Generalization

泛化是一种继承关系，表示一般和特殊关系，例如有动物类Animal，狗是动物的一种，狗类就继承了Animal。

泛化用带实线的空三角箭头表示，箭头指向父类

实现 Association

实现是类和接口的关系

实现用带虚线的空三角箭头表示，箭头指向接口

关联 Association

关联分为单向关联，双向关联，自身关联。也可以分为一对一关联和一对多关联

单向关联表示一个类持有另外一个类的属性，单向关联用一个带实线的箭头表示。箭头指向被关联者

上面的图表示学生有多个课程，课程被学生关联

双向关联就是两个类互相持有对方类型的属性，双向关联用一条不带箭头的实线表示

上面的图表示一个老师有多个学生，一个学生，他们互相关联

自身关联表示一个类与自己有关联，这个只是特殊的单向关联

聚合 Aggregation

聚合是整体和部分的关系，而且部分可以离开整体单独存在， 例如汽车和轮胎，轮胎脱离了汽车还可以存在

聚合用一边是箭头一边是空心菱形表示，箭头指向部分，空心菱形指向整体

组合 Composition

组合也是整体和部分的关系，但是和聚合不同的是，部分不可以离开整体单独存在，例如动物无法离开生态环境而存在。

组合用一边是箭头一边是实心菱形表示，箭头指向部分，实心菱形指向整体

依赖 Dependency

依赖是一种使用的关系，例如司机必须使用汽车，所以说司机依赖汽车

依赖用带箭头的虚线表示，箭头指向被使用者
