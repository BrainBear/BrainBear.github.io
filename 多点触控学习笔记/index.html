多点触控学习笔记

Posted on 2018-08-05

最近在些自定义控件的时候，遇到了多点触控的问题，所以今天来总结一下关于多点触控的知识。

不处理多点触控存在的问题

一般代码中就是处理触摸的ACTION_DOWN、ACTION_MOVE和ACTION_UP事件，在单点触控的情况下是没问题的，但是在多点触控的时候呢？我特意做了个画板demo：

逻辑很简单，就是画出触摸的轨迹，可以看到在多点触摸的情况下有几个问题：

  1. 第二手指的轨迹没有画出来
  2. 当第一根手指松开，第二跟手指没松开的情况下，轨迹会有跳跃

MotionEvent

Android把触摸输入都封装在MotionEvent中，我们先了解一下MotionEvent有什么方法：

方法 描述
getAction() 获取事件类型
getX() 获取事件在当前view的x坐标
getY() 获取事件在当前View的Y坐标
getRawX() 获取事件在屏幕的X坐标
getRawY() 获取事件在屏幕的Y坐标

上面的一些方法我们都很清楚了，单点触控也需要调用这些方法处理相关逻辑，下面的一些方法就和多点触控相关了：

方法 描述
getActionMasked() 获取事件类型
getActionIndex() 获取事件序号
getPointerCount() 获取事件有多少个触摸点
getPointerId(int pointerIndex) 通过事件序号来获取PointerId
findPointerIndex(int pointerId) 通过PointerId来获取事件序号

事件类型

我们先看看MotionEvent中有

从上面的两个表中我们看到有两个方法来获取事件类型：getAction()和getActionMark()，我们先看看他们的方法实现：

从代码上看ActionMark就是Action的低八位，这是因为Android是这么封装多点触控的：Action是一个int值，由ActionMark和ActionIndex组成，ActionMark和ActionIndex都占8位。ActionIndex占8-15位，ActionMark占0-7位。

也就是说ActionMark才是真正的事件类型，但是不是全部事件都有ActionIndex，ACTION_MOVE没有ActionIndex。

我们看看getActionIndex的源码：

MotionEvent提供了很多ActionMark，但是我们需要的只有：

事件类型 值 说明
ACTION_DOWN 0 第一个手指按下
ACTION_UP 1 最后一个手指按下
ACTION_MOVE 2 手指移动
ACTION_CANCEL 3 事件被拦截
ACTION_POINTER_DOWN 5 有新的手指按下
ACTION_POINTER_UP 6 有手指抬起，而且不是最后一个手指

举个例子，当有多个手指连续按下：

  1. 第一个手指按下 ActionMark = 0 ActionIndex = 0 Action = 0
  2. 第二个手指按下 ActionMark = 5 ActionIndex = 1 Action = 81
  3. 第三个手指按下 ActionMark = 5 ActionIndex = 2 Action = 82

ActionIndex和PoninterId

ActionIndex用来表示事件序号，但是在Move是没用的，那么我们怎么在滑动的时候判断哪个手指移动到哪里了呢？这个时候就需要PointerId了。在一个事件序列中，可以有多个手指触摸屏幕又松开屏幕，在这个过程中ActionIndex是会变化的，但是PointerId是不变的，PointerId用来标识一个手指的轨迹。
