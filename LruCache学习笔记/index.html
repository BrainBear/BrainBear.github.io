LruCache学习笔记

Posted on 2019-01-13

对于用户来说，空白的加载状态太影响体验了，产品设计的时候一般都会将上次加载的数据缓存下来，一些不变动的数据缓存之后下次使用就可以秒开，但是现实条件限制不可能将所有的数据都缓存下来，这就需要缓存算法。其中 LRU 算是最常用的算法了，在 Android 很多优秀的开源框架都用到了这种算法，例如：OkHttp、Glide 等。

LRU 算法

LRU 是 Least Recently Used 的缩写，简单来说就是找出最近最少使用的数据，它是基于这样一个前提：最近最少使用的数据，以后再用的可能性就越低。所以先丢弃这部分数据，让位置给新的数据。

LRU 算法比较简单，核心就是要找到最近最少使用的数据。既然是最少使用，那么我们每次将数据按使用时间来排序，例如构建一个队列，添加新的数据就添加到队尾，如果访问了队列中的数据，就将该数据移动到队尾。也就是说每次最新使用的数据就在对尾，那么队头的数据就是最近都没有使用过的，在新加入数据的时候，如果队列慢了，就将队头的数据移除。

因为对于缓存的数据，我们还要构建一个 key-value 的对应关系，方便查询缓存，在 java 中提供了一个类可以实现我们想要的功能：LinkedHashMap 。

LinkedHashMap 的构造方法提供了一个 accessOrder 的参数，如果传入 false，LinkedHashMap 内部是根据数据插入的顺序排序，但是如果传入 true 的话，就根据访问的顺序排序。

也就是说直接使用 LinkedHashMap 就可以提供了我们想要的数据结构，那么里面的是怎么实现的呢？
