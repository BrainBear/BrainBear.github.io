如果是分支比较多的时候，在 Java 中我们一般使用 switch ，但是在 Kotlin 中提供了增强版的 when 。

when 会将参数与所有分支条件按照顺序比较，直到某个分支满足条件，如果有多个条件执行同样的逻辑，可以使用 , 并列多个条件。when 既可以当做表达式也可以当成语句使用，如果 when 用于表达式，则必须要覆盖所有的条件，否则编译出错。

对于 Java 来说，switch 只支持特定的类型，而 when 支持任意表达式作为分支条件：

主构造函数

在 Java 中，一个类可以有多个构造函数， Kotlin 也一样，但是 Kotlin 中的构造函数分为主构造函数 和 次构造函数，一个 Kotlin 类只能有一个主构造函数，但是可以有多个次构造函数。主构造函数是类头的一部分，跟在类名后面：

如果主构造函数没有注解或者可见性修饰符修饰，就可以省略 constructor 关键字。

主构造函数没有代码逻辑，如果需要初始化，可以使用 init 初始化块。

init 代码块是按照在类的顺序来执行的，init 代码块可以访问在前面定义的属性，但是不可以访问在后面定义的属性，主构造函数的参数既可以在 init 中访问，也可以用于初始化类的属性。

重写 override

在 Kotlin 中子类也可以重写父类的方法，不同的是 Kotlin 类中的方法默认都是不能被重写的，需要显式声明 open 。子类重写的方法也必须使用 override 修饰，如果没有的话编译会出错。

因为属性实际上也是方法，所以在 Kotlin 中也可以重写方法。

抽象类

被 abstract 修饰的类称为抽象类，在抽象类中，有一些类成员可以在本类中声明为 abstract，就不需要再该类实现，而且交给子类来实现。被 abstract 修饰就默认是 open 的。

接口

Kotlin 的接口和 Java 8 中的接口类似，接口中有方法的声明，也可以有无状态的默认实现。可以有属性，但是隐式声明为抽象，或者提供访问器。

可见性修饰符

在软件项目开发中，代码封装是一个设计理念，最常用的方式就是通过设置不同的权限或者或可见性来实现，Kotlin 中有以下可见性修饰符：

  • private
  • protected
  • internal
  • public

对于类成员来说：

  • private 只能在该类内部可见
  • protected = private + 子类
  • internal 同一个模块内，这个我也不是很理解，暂且在 Android Studio 中是同一个 moduel
  • public 全部都可见
