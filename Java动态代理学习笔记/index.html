Java动态代理学习笔记

Posted on 2017-06-29

代理模式是一种常见的设计模式，通过代理可以为对象提供一个代理对象，并通过代理对象控制间接访问原对象，代理对象可以在执行原对象的操作前或者操作后执行自己的逻辑。

名词解释

  • 委托类与委托对象

    委托类是一个类，委托对象是委托类的实例，即上面描述的原对象，在本例中委托类是RealSubject.class

  • 代理类与代理对象

    代理类是一个类，代理对象是代理类的实例。

  • 动态代理与静态代理

    代理模式分为静态代理和动态代理，静态代理比较简单，所谓的静态是程序员在源代码中写好了代理类，代理类和委托类的关系在程序运行前就确定了。而动态代理中的代理类是在程序运行时JVM通过反射等机制动态生成的，源码中并不包含代理类，所以代理关系是在程序运行时确定的。

Java中的动态代理

Java中的动态代理主要涉及如下几个类:

java.lang.reflect.Proxy

Java 动态代理机制生成的所有动态代理类的父类，它提供了一组静态方法来为一组接口动态地生成代理类及其对象。主要有以下方法：

java.lang.reflect.InvocationHandler

这是调用处理器接口，它自定义了一个 invoke 方法，用于集中处理在动态代理类对象上的方法调用，通常在该方法中实现对委托类的代理访问。每次生成动态代理类对象时都要指定一个对应的调用处理器对象。

Java中实现动态代理的步骤

  • 定义一个接口，委托类和代理类都实现了该接口。在本例中接口为Subject.class
  • 定义一个处理类，该类实现了InvocationHandler接口，重写InvocationHandler接口的invoke方法实现自己的逻辑
  • 通过Proxy类的newProxyInstance方法获取动态代理对象，通过代理对象调用相应的方法

  1. 在工程项目右键选择Properties
  2. 在弹出框中选择Java Build Path
  3. 在右侧选择Librarise选项卡
  4. 点开JRE System Library，选中Access Rules
  5. 点击右侧的Edit
  6. 在弹出框中点击Add
  7. 在弹出框中Resolution选择Accessible，Rule Pattern中填入**
  8. 一路确定
